## Purpose

One-time onboarding that gathers clinical context and writes `.clinical-context.md` to the doctor's project root. All other skills check for this file and adapt accordingly.

## Step 1: Check for Existing Context

Silently scan the project root for `.clinical-context.md`.

- **If found:** Show the current preferences and ask: "Would you like to update your clinical preferences, or are these still correct?"
  - If update → proceed to Step 2, pre-filling known answers
  - If correct → confirm and exit: "Your preferences are up to date. They'll be applied to everything I build."
- **If not found:** Proceed to Step 2.

## Step 2: Gather Clinical Context

Ask these questions conversationally — never in a technical or survey-like tone. Adapt to the doctor's language (Spanish/English).

1. **Specialty:** "What is your medical specialty?" (e.g., internal medicine, pediatrics, emergency medicine, cardiology)
2. **Practice type:** "What type of practice do you work in?" (hospital / clinic / private practice / academic / mixed)
3. **Patient population:** "What patient population do you primarily treat?" (adults / pediatric / geriatric / neonatal / mixed)
4. **Guidelines:** "Are there specific clinical protocols or guidelines your institution follows?" (e.g., Surviving Sepsis Campaign, ADA diabetes guidelines, local institutional protocols)
5. **Units:** "What units do you prefer?" (metric/imperial, mg/dL vs mmol/L for glucose, Fahrenheit vs Celsius for temperature)

**Rules:**
- Accept short answers — don't force elaboration
- If the doctor says "defaults are fine" for any question, use sensible defaults (internal medicine, hospital, adults, standard guidelines, metric + mg/dL + Celsius)
- Never ask technical questions about code, components, or architecture

## Step 3: Write Context File

Write `.clinical-context.md` to the project root with this format:

```
# Clinical Context

> Auto-generated by /teach-clinical. Re-run to update.

## Specialty
{answer}

## Practice Type
{answer}

## Patient Population
{answer}

## Institutional Guidelines
{answer}

## Preferred Units
- Temperature: {Celsius/Fahrenheit}
- Glucose: {mg/dL or mmol/L}
- Weight/Height: {kg/cm or lbs/in}
- Blood pressure: mmHg (standard)
```

## Step 4: Confirm

Tell the doctor: "I've saved your clinical preferences. These will be applied to everything I build for you — from vital sign ranges to unit displays to clinical alerts. You can update them anytime with /teach-clinical."

## NEVER
- Ask technical questions (frameworks, databases, APIs)
- Require the doctor to answer every question — defaults are fine
- Overwrite an existing context file without asking first
- Store any patient-identifiable information in the context file
