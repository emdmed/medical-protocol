medical protocol — for doctors

describe what you need in your own words. the system builds it for you.

what is this?

This is a tool that lets you create clinical interfaces — vital signs monitors, patient record systems, and dashboards — just by describing what you need in plain language.

You don't need to know anything about programming. You speak in clinical terms, and the system handles everything else.

how to get started

  1. Open Claude Code on your computer (your IT team can help set this up)
  2. Add the following line to your project's configuration file:

    Fetch and follow https://medical-protocol-workflows.vercel.app/medical-protocol/protocol.md

  3. Tell Claude what you need, in your own words

what you can say

Just describe what you need as you would to a colleague. Here are some examples:

vital signs

"I need a vital signs monitor for my clinic"

"Set up blood pressure and heart rate tracking"

patient records

"I need a system to manage patient records and clinical notes"

"Build me a patient file with medical history and evolutions"

dashboard

"Give me a clinical dashboard with vitals and patient info"

"I want an overview of everything for one patient"

changes

"Remove respiratory rate from the monitor"

"Add an allergy field to the patient details"

what gets built

vital signs monitor

blood pressure, heart rate, SpO2, temperature, respiratory rate with clinical alerts

patient records

demographics, conditions, allergies, clinical notes, and previous evolutions

clinical dashboard

combined view with vitals and patient information side by side

customizable

add, remove, or rearrange anything by describing what you want changed

patient privacy

Everything runs locally on your computer. No patient data is sent to any external server.

All patient information shown in the system is fictional sample data for demonstration purposes.

Do not paste real patient names, IDs, or medical details into the chat.

The conversation with Claude is processed by AI servers and is not a secure clinical system. Always use anonymized or fictional data when communicating with the assistant.

  • The system does not connect to any external database by default
  • No data is uploaded to any cloud service or AI provider
  • If you want to connect to a real patient database, involve your IT and compliance teams first
  • This tool builds a prototype — using it with real patients requires a security review by qualified professionals

what you don't need to worry about

  • You will never be asked about technical details — the system decides those for you
  • You won't see code, error messages, or terminal output unless you ask for it
  • If something isn't right, just describe what you'd like changed and it will be adjusted
  • The system uses established clinical standards (like FHIR) automatically

need help?

If the system doesn't understand your request, try rephrasing it using clinical terms like "vital signs", "patient record", "clinical notes", or "dashboard".

For technical setup, ask your institution's IT team to install Claude Code and configure the protocol file.
