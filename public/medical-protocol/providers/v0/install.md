# Medical Protocol — Get Started

## Step 1: Open v0.dev

Go to [v0.dev](https://v0.dev) and start a new chat.

## Step 2: Describe what you need

Tell it what clinical tools you want, using the medical protocol as context. Here are some examples:

**Vital signs monitor:**
> Read the medical protocol at https://medical-protocol.vercel.app/medical-protocol/providers/v0/protocol.md — it has pre-built clinical components in a registry. Install and use them as-is, never rebuild them from scratch. I need a vital signs monitor for my clinic that tracks blood pressure, heart rate, and oxygen saturation.

**Clinical dashboard:**
> Read the medical protocol at https://medical-protocol.vercel.app/medical-protocol/providers/v0/protocol.md — it has pre-built clinical components in a registry. Install and use them as-is, never rebuild them from scratch. Build me a clinical dashboard with vital signs and a blood gas analyzer.

**Fluid balance tracker:**
> Read the medical protocol at https://medical-protocol.vercel.app/medical-protocol/providers/v0/protocol.md — it has pre-built clinical components in a registry. Install and use them as-is, never rebuild them from scratch. I need to track fluid intake and output for admitted patients.

**DKA monitoring:**
> Read the medical protocol at https://medical-protocol.vercel.app/medical-protocol/providers/v0/protocol.md — it has pre-built clinical components in a registry. Install and use them as-is, never rebuild them from scratch. I need a DKA monitoring dashboard that tracks glucose, ketones, potassium, and runs blood gas analysis on every reading.

**Full patient workspace:**
> Read the medical protocol at https://medical-protocol.vercel.app/medical-protocol/providers/v0/protocol.md — it has pre-built clinical components in a registry. Install and use them as-is, never rebuild them from scratch. I need a dashboard with vital signs, a BMI calculator, and a blood gas analyzer for my surgical patients.

## Step 3: Preview and deploy

You'll see a live preview of your interface. When it looks right, deploy it with one click — you'll get a URL you can open on any device.

## Available tools

| Tool | What it does |
|------|-------------|
| Vital signs | Blood pressure, heart rate, respiratory rate, temperature, SpO2 with clinical alerts |
| Blood gas analyzer | ABG interpretation with disorder detection, compensation, and anion gap |
| BMI calculator | Body mass index with imperial/metric toggle and category classification |
| Fluid balance | Intake/output tracker with insensible loss and net balance |
| PaFi calculator | PaO2/FiO2 ratio with ARDS severity classification |
| DKA monitoring | Hourly glucose, ketones, potassium, insulin, GCS with blood gas analysis |
| Dashboard | Combine any of the above into one view |

---

## Privacy

All patient data stays in your browser. The deployed interface hosts only the interface code — patient data is stored in your browser's local storage and is never uploaded, transmitted, or accessible to anyone but you.

Enter patient data directly into the interface, not into the chat.
